# Accessibility Guardian — 5-Minute Demo Script

## Objective
Show developers immediate value in under 5 minutes.

## Setup Before Recording
- Use a small sample project with intentional issues:
  - missing `alt`
  - skipped heading order (`h1` to `h3`)
  - ARIA misuse
- Ensure extension is installed and enabled
- Open VS Code Problems panel

---

## 0:00–0:30 — The Problem
**On screen:** problematic HTML file  
**Say:**
> Accessibility issues often slip into production because developers don’t see them while coding.  
> Accessibility Guardian brings those signals into VS Code early.

## 0:30–1:00 — Install
**On screen:** Marketplace page + install button  
**Say:**
> Install Accessibility Guardian from the VS Code Marketplace and open your project.

## 1:00–2:00 — Scan Active File
**On screen:** Command Palette → `Accessibility Guardian: Scan Active File`  
**Say:**
> Run a scan on the active file.  
> Findings appear in the Problems panel with line-level context.

Highlight 2–3 findings quickly.

## 2:00–3:00 — Fix One Issue
**On screen:** fix missing `alt` / ARIA misuse  
**Say:**
> Let’s fix one issue now and rerun the scan.

Rerun scan, show the issue disappearing.

## 3:00–4:00 — Deep Scan Workspace
**On screen:** `Accessibility Guardian: Deep Scan Workspace`  
**Say:**
> For broader coverage, run a deep workspace scan across project files.

Show multiple findings surfacing.

## 4:00–5:00 — Wrap + CTA
**On screen:** clean summary + Marketplace page  
**Say:**
> Accessibility Guardian helps teams catch accessibility and compliance risks before release.  
> Install it from the VS Code Marketplace and start scanning in minutes.

CTA link:
https://marketplace.visualstudio.com/items?itemName=EchoCoreLabs.accessibility-guardian
