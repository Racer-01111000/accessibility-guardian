# Accessibility Guardian — 30-Day Install Sprint (25 → 100)

## Goal
Build compounding discovery for Marketplace installs by combining:
1. Marketplace conversion improvements
2. GitHub trust signals
3. Weekly developer content distribution

---

## Weekly Targets

- **Week 1:** 25 installs
- **Week 2:** 45 installs
- **Week 3:** 70 installs
- **Week 4:** 100 installs

Primary KPI: installs/week  
Secondary KPI: listing conversion lift (views → installs), new ratings/reviews.

---

## Channel Split (Practical)

### 1) Marketplace (primary)
- Keep listing fresh with monthly releases
- Add screenshot set (4 images minimum)
- Keep title/description keyword-dense but readable
- Ask every happy user for a review in plain language

### 2) GitHub (trust + discovery)
- README clear, skimmable, and visual
- Pin one realistic demo GIF in README
- Add 3 issue templates: bug, rule request, false positive
- Use release notes that call out practical user impact

### 3) Developer communities (distribution)
- 1 post/week (Dev.to, Reddit, X, Indie Hackers)
- Topic-led content, not sales copy
- CTA: “Try it in VS Code Marketplace”

---

---

## Quality Guardrails
- No exaggerated claims (“guaranteed compliance”, “fully automated legal protection”).
- Focus on “helps detect”, “surfaces likely risks”, “supports shift-left workflows”.
- Keep tone practical and developer-first.

---

## Weekly Review (15 min)
Every 7 days, check:
- installs delta
- new reviews
- top referrer sources
- most common user feedback

Then update one of:
- listing copy
- screenshots
- roadmap priority

Compounding starts when every release and post points to one clear install path.
